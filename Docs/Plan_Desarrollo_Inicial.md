# Plan de Desarrollo Inicial

## 1. Análisis
- Definir el problema que resuelve la aplicación y a quién va dirigida.
- Elegir el MVP (Minimum Viable Product).
- Determinar las funcionalidades mínimas para la primera versión.

## 2. Selección técnica

### Framework
- Decidir el framework principal (UIKit).

### Arquitectura
- Seleccionar la arquitectura base.

## 3. Documentación de la aplicación
- Crear el PID.
- Crear bocetos.
- Describir el MVP.
- Elaborar diagramas.
- Registrar decisiones técnicas.
- Documentar arquitecura.

## 4. Configuración del repositorio
- Crear el repositorio.
- Crear el archivo README inicial.
- Configurar ramas.
- Preparar la rama principal (main).
- Añadir el archivo `.gitignore`.

## 5. Desarrollo
- Crear la estructura base del proyecto en Xcode.
- Preparar UI basica para testing manual.
- Crear ciclo completo de funcionamiento (ServiceApi, repository, viewModel...).
- Añadir control de errores personalizados.
- Mejorar UI añadiendo la segunda pantalla
- Programar el coordinatorApp.
- Finalizar UI


## 6. Testing
- Realizar tests manuales de flujo.
- Crear tests unitarios de ViewModels y componentes.

## 7. Mantenimiento
- Actualizar documentación.
- Solucionar bugs.
- Aplicar mejoras y actualizaciones.
